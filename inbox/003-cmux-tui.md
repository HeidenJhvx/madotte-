# cmux / 터미널 운영 전환 — 관제탑 인프라

- 무엇: cmux — Ghostty 기반 무료 오픈소스 네이티브 macOS 터미널. AI 에이전트 병렬 운용
  특화: 에이전트별 알림 링/사이드바, 세션 복원(재부팅 생존), 소켓 API(스크립트 제어),
  내장 브라우저, SSH·원격 tmux attach. 설치: brew install --cask cmux
- 왜 inbox: VS Code(페어코딩 모델) → 터미널 멀티플렉서(감독 모델) 전환은
  v0.4(무인 상시)·v0.5(병렬 멀티에이전트)의 전제 인프라. 소켓 API = 랄프루프/오케스트레이션의
  스크립트화 = 개입 횟수의 구조적 감소 경로.
- 시점: **v0.2 경계에서 도입** (사이클 중간 인터페이스 변경 금지 — 2·3조 교란 방지).
  v0.1의 밤에는 설치하지 않는다 (7조).
- 도입 절차: v0.1 종결 → 설치 → 1세션 동일 워크플로 재현 → v0.2 사이클부터 사용.
- 연계: cmux 조작용 Claude Code Skill 존재 (서브에이전트를 pane으로 가시화) —
  v0.5 오케스트레이션 참조. 단 에이전트에 터미널 타이핑 동급 권한을 주는 것이므로
  도입 시 Ch.13(툴 게이팅) 소환 트리거.
- 원격: VPS에선 tmux 병용 (cmux가 원격 tmux attach 지원) — v0.4 클라우드 계획과 접속.
