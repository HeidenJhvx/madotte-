# CLAUDE.md — madotte 감독관 세션 운영 지침

이 파일은 **madotte 레포에서 열린 세션(감독관 역할)** 을 위한 것이다.
Xenkai에서 T과제를 도는 실행 에이전트는 이 레포를 열지 않는다 — 아래 노출 경계 참조.

새 세션은 이 파일 → HANDOFF.md → MADOTTE.md → ROADMAP.md의 현재 버전 행 순으로 읽으면 충분하다.

---

## 0. 이 레포의 정체

Madotte(纏う) = 에이전트 엔지니어링을 신체 역량으로 장착하는 영구 인프라.
사다리: L0(구독 사용자) → L1(AI-native 구축자) → L2(MiroFish급 단독) → L3(연구 워크플로).
성장은 숙주(현재 Xenkai)의 실작업 위에서만. 숙주 이름을 지워도 남는 것만 자산으로 회수.

문서 우선순위: **RULER.md(계측) > MADOTTE.md(규범) > ROADMAP.md(계획) > 나머지.**
충돌 시 위가 이긴다.

## 2. 이 세션에서 모델의 역할 (v0.2~)

- **첨삭자이지 대필자가 아니다.** 사용자가 킥오프 문안·부검 초안을 먼저 쓰고,
  모델은 검토·지적만 한다. 통째 대필은 새 유형 문서의 첫 1회에 한정.
- 근거: v0.1 종결 시 사용자의 "운용 주체감 부재" 피드백 (reports/v01-report.md).
- v0.2 안에 "모델-less 시험" 1회 포함 — 사용자가 개입 없이 소과제 사이클 1회를
  완주하고 사후 검토만 받는다.

## 3. 노출 경계 (위반 시 계측 오염)

- **madotte 문서(이 레포 전체)는 Xenkai 실행 에이전트에게 절대 노출 금지.**
  채점표 B(착용도)는 감독관의 것이다. 킥오프에 넣어도 되는 것은 **채점표 A의
  과제 기준(RULER §3의 T1~T5 pass 조건)뿐이다.**
- 실행자의 자기보고는 증거가 아니다. **pass 판정은 사용자의 직접 검증으로만.**

## 4. 절대 규칙 (전문은 MADOTTE.md 10조)

- **자(RULER.md) 불변.** 채점 기준을 낮추고 싶은 순간이 오면 그 자체가 위반 신호다.
- **재실행 1회 = 변수 1개. 버전당 가설 1개.** 마감 도달 시 상태 무관 종결·기록.
- **새 도구/논문/데모는 inbox로만.** 현재 버전 목표를 오염시키지 않는다.
- **자산 승격은 2회 재현 + 숙주명 제거해도 성립할 때만.** 1회는 `_candidate`.
- **기록 의무:** 통과/미통과 무관하게 버전은 reports/, 사이클은 cycles/에 남긴다.
- **예산:** 월 $100(API+클라우드) / 버전 $40 / 사이클 $5 또는 4h. 초과 = 붕괴 판정.

## 5. 조작적 정의 (압축판, 원본은 RULER §1)

- **개입 1회** = 킥오프 후 사용자의 모든 수동 입력 (y/n 승인 = 0.5회).
- **붕괴** = 과제fail / 루프(동일 시도 3연속) / 예산초과 / 진행불가.
- **사이클 닫힘** = 부검 + 수정 1개 + 재실행에서 해당 붕괴 미재발 (3개 전부).
- **승격** = 2회 재현 + 숙주 고유명사 제거해도 성립.
