## 4. 차기 모델(Opus)의 역할 — v0.1과 달라진 것

- v0.1에서 Fable은 대필자였다. **v0.2부터는 첨삭자다.** 사용자가 킥오프 문안·부검
  초안을 먼저 쓰고, 모델은 검토·지적만 한다. 통째 대필은 새 유형 문서의 첫 1회만.
  (근거: v0.1 종결 시 사용자의 "운용 주체감 부재" 피드백 — reports/v01-report.md)
- v0.2 안에 "Fable-less 시험" 1회 포함: 사용자가 모델 개입 없이 소과제 사이클
  1회를 완주하고 사후 검토만 받는다. 운용감의 조작적 판정.

## 5. 절대 규칙 요약 (전문은 MADOTTE.md)

- 자(RULER.md) 불변. 채점 기준을 낮추고 싶은 순간이 오면 그게 위반 신호.
- 재실행 1회 = 변수 1개. 버전당 가설 1개. 마감 시 상태 무관 종결.
- madotte 문서는 실행 에이전트(Claude Code)에게 절대 노출 금지.
  채점표 B는 감독관의 것이다. 과제 기준(채점표 A)만 킥오프에 포함 가능.
- 실행자의 자기보고는 증거가 아니다. pass 판정은 사용자의 직접 검증으로만.
- 새 도구/논문/데모는 inbox로만. 현재 버전 목표 오염 금지.
- 예산: 월 $100(API+클라우드), 버전 $40, 사이클 $5 또는 4h.

## 6. 조작적 정의 압축판

개입 1회 = 킥오프 후 사용자의 모든 수동 입력 (y/n 승인 = 0.5).
붕괴 = 과제fail / 루프(동일 시도 3연속) / 예산초과 / 진행불가.
사이클 닫힘 = 부검 + 수정1개 + 재실행 미재발.
승격 = 2회 재현 + 숙주명 제거해도 성립.
