## 3. 부검 — "왜 됐는가" (5줄)

1. 킥오프에 정찰 보고서가 물려 있었다: 죽은 호스트/살아있는 호스트/원인 추측이
   문서화되어 있어 에이전트가 탐색 없이 곧장 수술 지점으로 갔다.
2. 즉 "정찰 산출물 = 컴파일된 컨텍스트"가 실행 중 개입 0의 유력 원인
   → v0.2 가설(부검/정찰→md 컴파일이 개입을 줄인다)의 **첫 증거 후보**.
3. 킥오프가 검증 방법(curl, mock-1~3 부재 확인)까지 지정해 에이전트가
   자가 검증 루프를 돌 수 있었다 — "완료 기준+검증법 내장"이 자기보고 품질을 올림.
4. 에이전트는 감사의 추측(env→죽은 호스트)을 실측으로 확정하고 최소 변경
   (env 1개 + 파일 1개)으로 수렴 — 스코프 제한 지시("나머지 5개 env 금지")가 유효했다.
5. 발생한 개입 2회는 전부 git 정책 결정(커밋/푸시) — 능력 문제가 아니라
   정책 위임 미비. 컨텍스트에 git 정책 1줄이 있었으면 0이 됐을 유형.
